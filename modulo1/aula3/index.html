Aula 3 - JavaScript

JavaScript é uma liguagem de programação multiparadigma, com ela pode-se escolher a forma como vai utilizar, usando a orientação a objetos, pode criar classes, usar métodos, pode ser visibilidade. Pode-se usar para programação funcional, atribuir funções dentro de variáveis, retornar funções na chamada de outra função.

O JavaScript atua de forma assíncrona, então pode deixar executando uma tarefa e ele te avisa quando estiver pronta. Podendo ter envios de mensagem e acessar informações no servidor. Esta linguagem não é totalmente pura em programação funcional, ou orientada a objetos, portanto pode usar o melhor de cada um, mas mesmo assim ela é estruturada.

JavaScript dá uma liberdade na hora de construir o código, porém uma desvantagem disso é que em alguns casos pode acontecer de fazer soluções muito complexas para problemas simples. Inicialmente ele foi desenvolvido para programação de páginas HTML, dando vida com comportamentos dinâmicos.

Por conta de JavaScript ser multiparadigma e ser interpretada, é uma linguagem de programação muito boa para navegadores, o código já vem do servidor e já é executado na página, fazendo toda a criação da lógica para a página. Há alguns anos surgiu o Node.JS, que é um interpretador de JavaScript, que faz com que ele seja executado em cima do sistema operacional, sendo executado do mesmo jeito que php, phyton, ruby.

Com o NodeJS é possivel construir o código back-end em JavaScript, assim como é construído em php no servidor, então pode programar em JavaScript no front-end e no back-end. E também é possivel usar JavaScript para consultas em bancos de dados noSQL, como é possível ver no MongoDB e CouchDB por exemplo, que utilizam o JavaScript como base para fazer consultas em suas bases de dados, esses bancos de dados são baseados em árvores (nós), e são basicamente um json, mais podereso e estruturado, e como JavaScript é a linguagem base para manipular dados desse tipo.



Histórico

Em 1993 mosaic era o navegador mais popular da Web. Que foi baseado no conceito do www de tim berners-lee, dominando os poucos usuários que tinha na época. Em 1994 surgiu o nestcape navigator, com a ideia de adicionar uma linguagem de programação dentro do navegador, por conta de ser até então códigos em HTML, hipertexto e algumas imagens.

Então tinhamos um padrão para um browser específico, o nestcape navigator, e se acessasse aquela página sem este browser, a página não iria funcionar, neste caso a linguagem usada é a skinner um dialeto do Lisp, é uma linguagem lógica. Com o passar do tempo, junto com a nestcape começou a concorrência com o internet explorer, para manter essa concorrência, o nestcape criou uma nova linguagem que fosse mais fácil programar, e por conta da popularidade do Java criou-se uma linguagem com o nome parecido e com a sintaxe parecida, que o protótipo ficou pronto em 10 dias.

Então em 1995 a Sun Microsystems criadora do Java, licenciou o uso do nome Java para a nestcape e assim surgiu o JavaScript. Porém é só o nome parecido, o JavaScript não usa base do Java, não se expira na sintaxe de classes. O Java que rodava no navegador, era um plugin que permitia a execução de Java nos navegadores, chamados de athletics, que possibilitava a criação de tela, usava-se muito Java nos navegadores por volta dos anos 2000, para internet backank, por conta que tinha problema de segurança básico, que era as pessoas instalando keyloggers, que tem a função de observar o teclado e guardar todas as teclas digitadas.

A solução era a inserção de dados, sem ter que digitar no teclado, então tinham teclados virtuais, parecido com o que se tem hoje nos smartphones, os teclados que aparecem quando precisa digitar algo, usava para digitar a senha do banco no navegador com os teclados virtuais. Quando acessava o netbank, para inserir a senha quando pedia, aparecia o teclado virtual em Java, para que não digitasse nada.

O problema do athletics em Java é que tinha muita falha de segurança, abria portas, e com isto no HTML5 já foi abolido, então não é bom usar esses plugins. O mesmo aconteceu com o flash nas suas animações ou o silverlight, o silverlight que veio para concorrer com o flash mais ele veio de uma microsoft muito adepta ao software proprietário, e as questões de licenciamento, e possuía um poder interessante de conseguir fazer streaming de vídeos sem que conseguissem piratear aqueles vídeos, portanto a tecnologia mais popular que a netflix utilizou foi o silverlight. E por isso precisa instalar o silverlight no navegador para acessar a netflix. Já o youtube usava flash para fazer seus primeiro players de vídeos.

O ECMAScript é um padrão de JavaScript, para que o interpretador JavaScript, o SpiderMonkey, que os navegadores possam entender melhor os códigos feitos em JavaScript, e também para que o programador também possa entender o código. E cada browser terá que implementar o interpretador JavaScript, o firefox tem um interpretador de JavaScript, também chamado de motor JavaScript, o google chrome tem o motor conhecido como V8 que é outro motor, a microsoft tentou fazer porém sem sucesso copiou o V8 e fez o microsoft edge (Chakra) baseado no chromium, que é a base do google chrome. O safari utiliza o JavaScriptCore.

ECMAScript foi criado para que não virasse bagunça, e cada navegador implementasse de um jeito, ele é uma especificação de como o JavaScript deve ser, quando um browser é compatível com o ECMAScript6, ele é compatível com uma das últimas versões do JavaScript.



JavaScript na Prática

Podemos inserir JavaScript de duas maneiras, no documento HTML, usando o tag <script> códigos JavaScript</script> ou usando a tag para chamar um arquivo com a extesão .js, que é a extensão para o JavaScript, a tag para chamar o arquivo é esta <script src="Endereço do arquivo.js"></script>

No método de tag <script> codigos JavaScript</script>, poderá ver alguns códigos com um atributo lang=" " para indicar JavaScript, ou a versão, mas hoje não faz muito sentido fazer isso, porque a linguagem principal que vai rodar no navegador será JavaScript. Antigamente era necessário, poís tinhamos o visual basic script, o vb script, o coffeeScript,o JavaScript, então tinha-se varios motores diferentes para estas linguagens.

Esses dois métodos podem ser colocados em qualquer lugar do documento HTML, tem pessoas que colocam no início da página, tem pessoas que colocam no final da página. Mas depende dos casos se o script tiver poucos códigos que tenham poucos processamentos, não fará diferença o local, caso tenha arquivos maiores melhor opção é colocá-los no final para que carregue todo o documento e todo o estilo primeiro para que a página não fique desconfigurada.

O JavaScript ele é executado a partir do momento que ele é inserido, por isso temos que tomar cuidado, tem as técnicas para dizer como é o comportamento dele colocando um atributo na tag. Com o atributo, é como falar para o script rodar assíncrono, ou quando a página toda carregar. Em geral o navegador pega o arquivo HTML, ele começa a interpretar linha a linha, quando chegar o script, faz uma requisição no servidor pega o arquivo JavaScrip, e vai executá-lo naquele momento.

Caso seu arquivo JavaScript estiver referenciando algum elemento que irá ser criado embaixo desse código, então vai dar erro, não vai executar o código, e vai falar que não existe esse elemento. O melhor é carregar todas as funções e depois as executar, para evitar desincronização no momento de carregar o documento.

Dentro das limitações do navegador, com JavaScript é possivel criar o que quiser, ele é uma linguaguem de programação. Um exemplo do que pode ser feito com ele é manipular a janela do navegador, mas hoje em dia alguns navegadores bloqueiam por conta dos spmmers, veja agora os exemplos para abrir e fechar uma janela:

  • Abrir uma Janela:

    • window.open("https://www.google.com.br");

  • Fechar uma Janela:

    • var w = window.open("https://www.google.com.br");

    w.close();

Esse primeiro comando abre uma nova aba, esse comando window.open() foi um dos mais famoso que tinham antigamente, por conta dos popups, quando abria uma página, os popups não abria dentro da página, eles abriam outra janela, não tinha nem abas nesta época.

Além de passar url neste comando, pode passar parâmetros para construir a página, como não poder ter barra de navegação ou tamanho fixo. Em JavaScript podemos usar o var nome; para definir uma variavel. Como foi dito esta técnica são bloqueadas por bloqueadores de popups.

Podemos também controlar o documento, que no caso é o documento HTML, já o comando window é a aba na qual o documento está aberto. No documento conseguimos manipular os elemento ou atributos ou até mesmo o estilo do documento aberto. veja exemplos:

  • Alterar o conteúdo:

    • document.getElementById("demo").innerHTML = "Olá JavaScript";

  • Alterar atributo:

    • document.getElementById("minhaImagem").src = "pic_bulboff.gif";


  • Alterar o Estilo:

    • document.getElementById("demo").style.display = "none";

Como podemos ver podemos mudar o texto do elemento, ou também mudar o src das imagens, e até mesmo construir todo o CSS com JavaScript, porém isso não é uma boa ideia. Mas tem algumas técnicas, uns frameworks para programar a interface com JavaScript e não cria css.
